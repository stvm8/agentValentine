#CTF #Methodology [2026-04-05] Avoid rabbit-hole local privesc enumeration (SUID, sudo -l, capabilities, systemd) if credential/password vectors are available—password reuse is often simpler than kernel exploits or capability abuse.
#ASREPRoast #BloodHound #ACL #DENY [2026-04-08] Issue: BloodHound reports AddSelf/AddMember edges but does NOT parse DENY ACEs in DACL — always read raw DACL (nTSecurityDescriptor) to verify. ACCESS_DENIED_ACE evaluated before ACCESS_ALLOWED_ACE in Windows. -> Solution: For any BloodHound ACL edge that fails, dump raw DACL with impacket + check for preceding DENY ACEs with same or broader mask.
#RBCD #UserAccountRBCD #S4U2Self #ForceForwardable #U2U #SessionKeyNTHash #impacket #HTB [2026-04-10] Issue: RBCD attack with user account (no SPN, MAQ=0). Standard S4U2Self fails (no SPN). U2U works for S4U2Self but force-forwardable fails because U2U ticket encrypted with TGT session key not long-term account key. Solution: (1) getTGT.py to get RC4 TGT, (2) describeTicket.py to extract 16-byte RC4 session key, (3) changepasswd.py -newhashes to set that session key AS the account NT hash, (4) getST.py -u2u -force-forwardable -hashes :SESSION_KEY - now force-forwardable can decrypt U2U ticket because account key = session key, (5) S4U2Proxy succeeds -> DA ticket -> secretsdump -> root
#SSHCert #doas #OpenBSD #PassTheHash #SMB #PTH #CertificateAuthority #LegacySSH #PubkeyAcceptedKeyTypes [2026-04-16] Issue: Modern SSH client drops ssh-rsa cert by default; OpenSSH 7.x only supports ssh-rsa-cert -> Solution: Add -o PubkeyAcceptedKeyTypes=+ssh-rsa-cert-v01@openssh.com when targeting old OpenSSH versions
#SSHCert #doas #OpenBSD #userca #AuthorizedPrincipalsCommand #SSHCAAbuse [2026-04-16] Issue: doas nopass <user> cmd /usr/bin/ssh-keygen -> can sign certs as that user's CA -> Solution: check TrustedUserCAKeys + AuthorizedPrincipalsCommand to determine principal needed; sign attacker pubkey with that principal
#PTH #SMB #LDAP #sambaNTPassword #PassTheHash #OpenBSD #NTHash #CredentialReuse [2026-04-16] Issue: LDAP anon bind exposes sambaNTPassword attribute -> Solution: Always extract sambaNTPassword from LDAP dumps; use with smbclient --pw-nt-hash or impacket for PTH without cracking
#PPK #PuTTY #SSHKey #KeyConversion #Paramiko #puttygen [2026-04-16] Issue: puttygen not installed on attack box; PPK v2 not directly loadable by paramiko RSAKey.from_private_key_file -> Solution: Parse PPK v2 manually (base64-decode Public-Lines/Private-Lines, extract RSA components, reconstruct via cryptography library) or install putty-tools
#AuthorizedPrincipalsCommand #SSHAuth #WebEndpoint #HTTPsshauth [2026-04-16] Issue: sshd AuthorizedPrincipalsCommand can point to an HTTP endpoint that controls who can log in -> Solution: Always curl the sshauth endpoint for the target user (type=principals&username=root) to find the required cert principal before signing
#MSSQL #ERRORLOG #CredentialLeak #PasswordInUsername #LateralMovement #CTF #sequel [2026-04-16] Issue: MSSQL ERRORLOG.BAK contains failed login attempts — user typed password in username field (Ryan.Cooper attempted login with password 'NuclearMosquito3') -> Solution: Always read C:\SQLServer\Logs\ERRORLOG.BAK and C:\Program Files\Microsoft SQL Server\*\MSSQL\Log\ERRORLOG* after MSSQL foothold; failed logins leak plaintext passwords typed in wrong field
#ADCS #ESC1 #ClockSkew #Kerberos #PKINIT #certipy #NTP [2026-04-16] Issue: certipy auth fails with KRB_AP_ERR_SKEW when DC clock skew > 5min -> Solution: sudo timedatectl set-ntp false && sudo date -s '<DC_TIME>' before certipy auth; re-enable NTP after. DC time from nmap clock-skew field or nxc SMB banner
